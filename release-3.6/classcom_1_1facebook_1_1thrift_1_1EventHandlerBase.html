ContextStack com.facebook.thrift.EventHandlerBase.getContextStack ( )

These functions are only used in the client handler implementation.

The server process functions maintain ContextStack on the stack and binds ctx in to the async calls.

Clients are not thread safe, so using a member variable is okay. Client send_ and recv_ functions contain parameters based off of the function call, and adding a parameter there would change the function signature enough that other thrift users might break.

The generated code should be the ONLY user of s_. All other functions should just use the ContextStack parameter.
